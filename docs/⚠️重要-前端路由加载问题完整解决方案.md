## 🏗️ 核心架构设计

### 标准流程

```
用户登录成功
  ↓
保存 Token → 跳转到首页 (/dashboard)
  ↓
路由守卫检测到权限未加载
  ↓
调用 permissionStore.fetchUserPermissions()
  ↓
从缓存/后端获取菜单树和权限
  ↓
注册动态路由到 router (router.addRoute)
  ↓
return { ...to, replace: true }  ← 关键！重新导航
  ↓
路由重新匹配，BasicLayout 组件挂载
  ↓
watch 监听 isLoaded → 生成菜单
  ↓
✅ 菜单显示成功
```

### 关键文件

- `src/utils/permission.js` - 路由守卫逻辑
- `src/stores/modules/permission.js` - 权限状态管理
- `src/stores/modules/menu.js` - 菜单数据管理
- `src/layouts/BasicLayout.vue` - 布局组件（监听权限加载）
- `src/router/guard.js` - 路由守卫配置

## 🐛 历史问题汇总

### 问题 1：权限加载成功但菜单不显示

**现象**：

- 控制台显示权限加载成功
- 路由构建成功（16个路由）
- 侧边栏显示"菜单数据为空"或一直显示"加载中"

**根本原因**：

```javascript
// ❌ 错误做法：直接返回 true
return true // 路由不重新匹配，动态路由不生效

// ✅ 正确做法：重新导航触发路由匹配
return { ...to, replace: true } // 触发路由重新匹配
```

**解决方案**：

1. `permission.js` 中使用 `return { ...to, replace: true }`
2. `BasicLayout.vue` 的 `onMounted` 中增加检查逻辑
3. 使用 `watch` + `immediate: true` 双重保险

### 问题 2：路由死循环

**现象**：

- 页面无限刷新
- 控制台出现 "Maximum call stack size exceeded"
- 路由守卫被反复调用

**根本原因**：

- 路由守卫中使用了 `replace: true` 但没有正确的终止条件
- `isPermissionLoaded` 状态判断不准确
- 重复调用 `fetchUserPermissions()`

**解决方案**：

```javascript
// permission.js 中的防重复加载机制
async function fetchUserPermissions() {
  // ✅ 防止重复加载：如果已加载，直接返回
  if (isPermissionLoaded.value === true) {
    console.log('✅ 权限已加载，跳过')
    return true
  }

  // ✅ 如果正在加载，返回缓存的 Promise（防止并发请求）
  if (loadingPromise) {
    console.log('🔄 权限加载中，等待完成...')
    return loadingPromise
  }

  // 创建新的加载 Promise
  loadingPromise = (async () => {
    // ... 加载逻辑
  })()

  return loadingPromise
}
```

### 问题 3：刷新页面后菜单丢失

**现象**：

- 首次登录菜单正常
- 刷新页面后菜单消失
- 路由变成 404

**根本原因**：

- 动态路由在刷新后会丢失
- 没有使用 localStorage 缓存
- 路由守卫没有重新注册路由

**解决方案**：

```javascript
// permission.js 中使用 localStorage 缓存
async function fetchUserPermissions() {
  // ✅ 优先从缓存获取
  const cachedMenuTree = localStorage.getItem('menuTree')
  const cachedPermissions = localStorage.getItem('permissions')

  if (cachedMenuTree && cachedPermissions) {
    console.log(' 使用缓存的菜单和权限')
    const menuTreeData = JSON.parse(cachedMenuTree)
    const permissionData = JSON.parse(cachedPermissions)

    setMenuTree(menuTreeData)
    menuStore.setMenuFromTree(menuTreeData)
    setPermissions(permissionData.map((p) => p.code))
    await registerDynamicRoutes(menuTreeData)
    isPermissionLoaded.value = true
    setLoaded()

    // ✅ 后台异步刷新缓存
    refreshPermissionsCache()
    return true
  }

  // 无缓存才从后端获取
  // ...
}
```

### 问题 4：BasicLayout 组件 watch 不触发

**现象**：

- 权限加载成功
- 但 BasicLayout 的 watch 没有执行
- 菜单数据没有生成

**根本原因**：

- BasicLayout 组件挂载时机晚于权限加载
- watch 的 `immediate: true` 在权限未加载时执行，拿到空数据
- 组件挂载后 watch 不再触发

**解决方案**：

```javascript
// BasicLayout.vue 中使用 onMounted + watch 双重保险
onMounted(() => {
  // ✅ 组件挂载时立即检查
  if (
    permissionStore.isLoaded &&
    permissionStore.menuTree?.length > 0 &&
    menuStore.menuOptions?.length === 0
  ) {
    console.log('⚡ 权限已加载但菜单为空，强制重新生成...')
    menuStore.setMenuFromTree(permissionStore.menuTree)
  }
})

watch(
  () => permissionStore.isLoaded,
  async (loaded) => {
    if (loaded === true) {
      await nextTick()
      if (permissionStore.menuTree?.length > 0) {
        menuStore.setMenuFromTree(permissionStore.menuTree)
      }
    }
  },
  { immediate: true }, // ✅ 立即执行一次
)
```

### 问题 5：菜单数据字段名不匹配

**现象**：

- 后端返回数据正常
- 前端菜单构建失败
- 控制台报错 "Cannot read property 'menuName' of undefined"

**根本原因**：

- 后端使用 `snake_case`（如 `menu_name`）
- 前端使用 `camelCase`（如 `menuName`）
- 字段名不匹配导致数据为空

## ✅ 解决方案总结

### 1. 路由守卫核心逻辑

```javascript
// src/utils/permission.js
export async function permissionGuard({ to, userStore, permissionStore, ... }) {
  const hasToken = userStore.token || localStorage.getItem('access_token')

  // 1. 未登录访问非公开页面 → 跳转登录
  if (!hasToken && !publicPaths.includes(to.path)) {
    return loginPath
  }

  // 2. 已登录访问登录页 → 跳转首页
  if (hasToken && to.path === loginPath) {
    return { path: '/dashboard', replace: true }
  }

  // 3. 权限正在加载中 → 直接放行
  if (hasToken && permissionStore.isLoading && !publicPaths.includes(to.path)) {
    return true
  }

  // 4. 已登录但权限未加载 → 加载权限
  if (hasToken && permissionStore.isNotLoaded && !publicPaths.includes(to.path)) {
    try {
      if (permissionStore.isLoading) {
        return true  // 已在加载中，直接放行
      }

      if (!userStore.userInfo?.user_id) {
        const res = await getCurrentUser()
        userStore.setUserInfo(res.data)
      }

      await permissionStore.fetchUserPermissions()

      // ✅ 关键：使用 replace: true 重新触发路由匹配
      return { ...to, replace: true }
    } catch (error) {
      permissionStore.setFailed()
      return true  // 降级放行
    }
  }

  return true
}
```

### 2. 权限 Store 防重复加载

```javascript
// src/stores/modules/permission.js
let loadingPromise = null // ✅ Promise 缓存，防止并发请求

async function fetchUserPermissions() {
  // ✅ 防止重复加载
  if (isPermissionLoaded.value === true) {
    return true
  }

  if (loadingPromise) {
    return loadingPromise
  }

  loadingPromise = (async () => {
    try {
      setLoading()

      // ✅ 优先使用缓存
      const cachedMenuTree = localStorage.getItem('menuTree')
      if (cachedMenuTree) {
        const menuTreeData = JSON.parse(cachedMenuTree)
        setMenuTree(menuTreeData)
        menuStore.setMenuFromTree(menuTreeData)
        await registerDynamicRoutes(menuTreeData)
        setLoaded()
        refreshPermissionsCache() // ✅ 后台刷新
        return true
      }

      // 从后端获取
      const [menuTreeRes, permissionsRes] = await Promise.all([getMenuTree(), getMyPermissions()])

      setMenuTree(menuTreeRes.data)
      menuStore.setMenuFromTree(menuTreeRes.data)
      await registerDynamicRoutes(menuTreeRes.data)
      setLoaded()

      return true
    } finally {
      loadingPromise = null // ✅ 清除 Promise 缓存
    }
  })()

  return loadingPromise
}
```

## 最佳实践

### 1. 路由守卫规范

- ✅ 使用 `return { ...to, replace: true }` 重新导航
- ✅ 使用 `isPermissionLoaded` 状态防止重复加载
- ✅ 使用 `loadingPromise` 缓存防止并发请求
- ✅ 使用 localStorage 缓存避免刷新白屏
- ❌ 不要使用 `return true`（动态路由不生效）
- ❌ 不要在登录页加载菜单（在路由守卫中加载）

### 2. 权限 Store 规范

- ✅ 维护三个状态：`false`（未加载）、`'loading'`（加载中）、`true`（已加载）
- ✅ 提供 `setLoading()`、`setLoaded()`、`setFailed()` 方法
- ✅ 优先使用缓存，后台异步刷新
- ✅ 退出登录时清除缓存和状态

### 3. 菜单生成规范

- ✅ 兼容 `snake_case` 和 `camelCase` 字段
- ✅ 过滤按钮类型（F）、隐藏菜单（visible='1'）、停用菜单（status='1'）
- ✅ 使用 `onMounted` + `watch` 双重保险
- ✅ 添加详细调试日志

### 4. 动态路由注册规范

- ✅ 先删除旧路由再注册新路由
- ✅ 过滤掉 `path: null` 的父菜单
- ✅ 最后注册通配路由（404）
- ✅ 使用 `router.addRoute('Layout', route)` 添加到 Layout 子路由
